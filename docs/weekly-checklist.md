
# Checklist Semanal de Uso do Repositório

## Objetivo
Este checklist existe para orientar o uso prático do repositório na rotina semanal de acompanhamento de projetos, aderência, tratativas e comunicação com liderança.

A ideia é transformar o repositório em um apoio recorrente para:
- leitura do cenário
- identificação de desvios
- priorização
- definição de ações
- consolidação semanal
- atualização para liderança

---

## Visão geral da semana
A lógica semanal de uso do repositório deve seguir este fluxo:

1. Ler o cenário
2. Identificar os desvios
3. Priorizar
4. Definir ações
5. Consolidar a semana
6. Preparar a liderança

---

# Segunda-feira — leitura inicial da semana

## Objetivo
Entender como a semana começa e onde estão os principais pontos de atenção.

## O que fazer
- [ ] Abrir os indicadores principais
- [ ] Ver aderência de projetos
- [ ] Ver aderência física
- [ ] Ver aderência financeira
- [ ] Ver programação semanal aderente
- [ ] Ver FMDs de projetos
- [ ] Identificar os projetos ou frentes mais críticas
- [ ] Listar os principais desvios observados

## Arquivos do repositório para usar
- [ ] `docs/workflow-weekly-routine.md`
- [ ] `docs/metric-definitions.md`

## Resultado esperado
- [ ] visão geral da semana
- [ ] lista inicial de desvios
- [ ] prioridades preliminares

---

# Terça-feira — análise de causa e priorização

## Objetivo
Entender melhor os desvios e separar o que é realmente crítico.

## O que fazer
- [ ] Revisar os desvios identificados
- [ ] Classificar cada desvio em alta, média ou baixa prioridade
- [ ] Identificar a causa provável de cada desvio
- [ ] Verificar se o problema está ligado a:
  - [ ] campo
  - [ ] planejamento
  - [ ] engenharia
  - [ ] gestão de contratos
  - [ ] medição
  - [ ] segurança
  - [ ] prontidão
  - [ ] dependência externa
- [ ] Identificar o que precisa de ação imediata
- [ ] Identificar o que precisa de escalonamento

## Arquivos do repositório para usar
- [ ] `docs/workflow-weekly-routine.md`
- [ ] `docs/output-standards.md`

## Resultado esperado
- [ ] prioridades definidas
- [ ] causas prováveis mapeadas
- [ ] base pronta para plano de ação

---

# Quarta-feira — plano de ação

## Objetivo
Transformar os principais desvios da semana em ações claras e acompanháveis.

## O que fazer
Para cada desvio importante:
- [ ] Registrar o problema
- [ ] Registrar a evidência
- [ ] Registrar a causa provável
- [ ] Registrar o impacto
- [ ] Definir a ação
- [ ] Definir responsável
- [ ] Definir prazo
- [ ] Definir prioridade
- [ ] Registrar status

## Arquivo do repositório para usar
- [ ] `templates/weekly_action_plan_template.md`

## Resultado esperado
- [ ] plano de ação semanal preenchido
- [ ] responsáveis e prazos definidos
- [ ] visão clara do que deve ser acompanhado

---

# Quinta-feira — consolidado da semana

## Objetivo
Juntar a leitura da semana em um resumo estruturado e reaproveitável.

## O que fazer
- [ ] Consolidar a visão geral do período
- [ ] Registrar os principais desvios
- [ ] Listar os projetos ou frentes críticas
- [ ] Consolidar os riscos da semana
- [ ] Registrar ações em andamento
- [ ] Registrar próximos passos
- [ ] Identificar o que precisa de apoio da liderança

## Arquivo do repositório para usar
- [ ] `templates/weekly-status-summary.md`

## Resultado esperado
- [ ] resumo semanal consolidado
- [ ] leitura executiva da semana
- [ ] base pronta para alinhamento

---

# Sexta-feira — atualização para liderança

## Objetivo
Transformar o consolidado da semana em uma mensagem executiva clara.

## O que fazer
- [ ] Preparar visão geral do cenário
- [ ] Destacar os principais desvios
- [ ] Explicar impactos
- [ ] Registrar ações em curso
- [ ] Destacar riscos
- [ ] Indicar apoio ou decisão esperada
- [ ] Fechar com próximos passos

## Arquivo do repositório para usar
- [ ] `templates/leadership_update_outline.md`

## Resultado esperado
- [ ] atualização para liderança
- [ ] texto para e-mail ou reunião
- [ ] direcionamento da próxima semana

# Checklist rápido por arquivo

## `docs/context.md`
Use quando precisar:
- [ ] relembrar o foco do seu trabalho
- [ ] manter coerência com seu contexto
- [ ] alinhar a lógica da análise com sua realidade

## `docs/metric-definitions.md`
Use quando precisar:
- [ ] interpretar aderência de projetos
- [ ] interpretar aderência física
- [ ] interpretar aderência financeira
- [ ] interpretar programação semanal
- [ ] interpretar FMDs

## `docs/output-standards.md`
Use quando quiser verificar se sua saída está:
- [ ] clara
- [ ] estruturada
- [ ] com fato, análise, impacto e ação
- [ ] sem ficar vaga ou genérica

## `docs/workflow-weekly-routine.md`
Use como:
- [ ] passo a passo semanal
- [ ] guia de leitura
- [ ] roteiro para sair do indicador até a ação

## `templates/weekly_action_plan_template.md`
Use quando precisar:
- [ ] organizar tratativas
- [ ] registrar responsáveis
- [ ] controlar prazos
- [ ] acompanhar ações por desvio

## `templates/weekly-status-summary.md`
Use quando precisar:
- [ ] consolidar a semana
- [ ] preparar reunião
- [ ] organizar os principais pontos
- [ ] fazer leitura gerencial

## `templates/leadership_update_outline.md`
Use quando precisar:
- [ ] preparar fala para liderança
- [ ] escrever e-mail executivo
- [ ] estruturar atualização de status
- [ ] resumir cenário com risco e ação
